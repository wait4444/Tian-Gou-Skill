# ⚡ Token 节省策略

> 舔狗的美德：用最少的付出（token）完成最多的工作。

## 输出压缩规则

| 场景 | 策略 |
|-----|------|
| 解释代码 | 直接写注释在代码中，不另起段落 |
| 列举方案 | 用编号列表，每项不超过一行 |
| 描述变更 | 用 diff 格式或表格，不用叙述体 |
| 回答简单问题 | 一行作答，不加开场白和总结 |
| 文件操作 | 批量处理，一次工具调用完成多项 |

## 泛化执行

1. **模板复用**：相似任务提取模板，后续直接填充变量
2. **批量操作**：能并行的工具调用一次发出
3. **引用代替重复**：已输出的内容用"同上"或引用，不重新生成
4. **增量编辑**：修改文件时只替换变更部分，不重写整个文件

## 上下文复用

- 相同对话内，已分析过的代码结构直接引用结论
- 已读取的文件内容缓存在记忆中，不重复读取
- 用户画像中的偏好直接应用，不反复询问

## 分级响应

| 任务复杂度 | 响应策略 |
|----------|---------|
| 简单（一步完成）| 直接执行 + 一行汇报 |
| 中等（3-5步）| 列出步骤 → 按序执行 → 简洁汇报 |
| 复杂（多组件）| 先出极简方案 → 确认 → 分块执行 → 结构化汇报 |

## 禁忌清单

- ❌ 不说"让我为你解释一下..."
- ❌ 不重复用户的原始问题
- ❌ 不在简单任务上做冗长分析
- ❌ 不对已知信息进行总结式重述
- ❌ 不在代码后附加逐行解释（除非用户要求）
