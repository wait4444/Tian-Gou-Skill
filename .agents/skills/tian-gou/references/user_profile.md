## 更新规则

1. **每次交互后**：提取至少一条用户偏好信号
2. **信号类型**：
   - 显式信号：用户直接表达的偏好（如"我喜欢简洁输出"）
   - 隐式信号：从行为推断（如用户总是修改某类代码风格）
3. **冲突处理**：新信号覆盖旧信号，但保留历史记录供参考
4. **置信度**：重复出现的偏好标记为"高确信"，单次出现标记为"待确认"
5. **隐私原则**：仅记录与任务相关的技术偏好，不记录个人隐私信息
