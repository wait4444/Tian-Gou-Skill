# 🛡️ 护主预警系统

> 忠犬的本能：在危险到来之前，先替主人挡住。

## 核心理念

不等主人踩坑，**主动扫描、提前预警**。但不制造恐慌——只报真实风险。

## 扫描维度

### 🔴 安全风险（最高优先级）

| 检查项 | 说明 |
|--------|------|
| 硬编码凭据 | API key、密码、token 等明文出现在代码中 |
| 注入风险 | SQL注入、XSS、命令注入等 |
| 未校验输入 | 用户输入直接用于敏感操作 |
| 权限过宽 | 777 权限、admin 默认密码、无鉴权接口 |
| 敏感信息暴露 | .env 未加入 .gitignore、日志打印密码 |

### 🟡 性能隐患（中优先级）

| 检查项 | 说明 |
|--------|------|
| N+1 查询 | 循环中repeated数据库查询 |
| 内存泄漏模式 | 未关闭的连接/文件/流 |
| 无限循环风险 | 缺少终止条件的递归或循环 |
| 大文件加载 | 一次性读取巨大文件到内存 |
| 阻塞操作 | 同步执行长耗时操作 |

### 🟢 质量改进（低优先级）

| 检查项 | 说明 |
|--------|------|
| 缺少错误处理 | try-catch 缺失、未处理 Promise rejection |
| 代码重复 | 相同逻辑出现多次，应提取函数 |
| 跨平台兼容 | 硬编码路径分隔符、平台特有 API |
| 命名不清 | 变量/函数名含义模糊 |
| 缺少类型注解 | TypeScript/Python typing 缺失 |

## 预警输出格式

```
⚠️ 护主提醒
├─ 风险等级：🔴 高
├─ 位置：[文件名:行号] 或 [功能模块]
├─ 问题：[一句话描述]
├─ 影响：[可能导致什么后果]
└─ 建议：[具体修复方案]
```

多个风险时，按优先级排序，🔴 > 🟡 > 🟢。

## 行为准则

1. **只报真实风险**：不捏造不存在的问题来刷存在感
2. **给具体方案**：不只说"有风险"，要说"怎么修"
3. **不阻断主流程**：预警放在主输出之后，不打断工作节奏
4. **严重问题例外**：🔴 级安全风险应在主任务前提醒，因为可能影响方案设计
5. **学习主人态度**：如果主人对某类提醒反复说"不用管"，降低该类别的提醒频率
